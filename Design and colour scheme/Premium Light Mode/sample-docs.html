Core concepts

Understanding the signal architecture

Last updated: 24 Mar 2026 · 8 min read

The signal architecture is built on the principle that data should flow with minimal friction from source to insight. Rather than forcing analysts through rigid pipelines, the system adapts to how professionals actually think — in loops, tangents, and progressive refinement.

Key principle

Every signal carries provenance metadata. You can always trace a conclusion back to its raw inputs — and the system makes this effortless rather than burdensome.

At the foundation level, signals are classified into three tiers based on their confidence interval and data lineage. This classification happens automatically, but can be manually overridden by senior analysts when domain knowledge supersedes statistical confidence.

Signal classification

The three-tier system uses a combination of source reliability scoring, temporal decay functions, and cross-reference validation to assign each signal a classification level. Tier 1 signals (designated T1) have the highest confidence and derive from primary sources with validated provenance.

// Signal classification pipeline const classifySignal = (raw) => { const confidence = calculateConfidence(raw); const decay = temporalDecay(raw.timestamp, 0.94); if (confidence * decay > 0.85) return 'T1'; if (confidence * decay > 0.60) return 'T2'; return 'T3'; };

The decay factor of 0.94 means signals lose roughly 6% of their confidence score per period. For market-sensitive data, this window can be tightened to 0.88 — reflecting the faster obsolescence of financial intelligence.

Important

Manual overrides on signal classification are logged and audited. The system maintains a complete override history for compliance review.

Pipeline integration

Signals flow through a composable pipeline where each stage can transform, enrich, or filter the data stream. The pipeline supports both synchronous processing for real-time dashboards and asynchronous batch processing for historical analysis.

Real-time Batch processing Hybrid mode

The hybrid mode is particularly useful during market events, where you want real-time signal flow augmented by historical context. The system automatically switches between modes based on signal volume thresholds, though this behaviour can be configured per-workspace.
