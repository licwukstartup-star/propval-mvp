02 — Version Control
📦
GitHub
Central repository storing all code with full change history. Every commit is tracked. Triggers automatic deployments to Vercel and Render when code is pushed. Enables rollback to any previous state.
source of truth · webhooks
↓ webhook triggers auto-deploy ↓
03 — Hosting & Infrastructure
🌐
Vercel
Frontend hosting platform. Deploys your Next.js app globally. Handles CDN, SSL, preview deployments per branch, and will serve your custom domain.
frontend host · CDN · edge
🗄️
Supabase
Postgres database + auth + real-time subscriptions + storage. Stores cases, comparables, EPC data, user accounts. RLS policies enforce multi-tenant data isolation.
database · auth · RLS · API
🔧
Render
Backend server hosting. Runs the comparable selection engine, external API orchestration, data enrichment pipelines, and any long-running processing tasks.
backend host · API server
↕ REST / real-time connections ↕

07 — Feedback Loops & Key Connections

↻ System Feedback Loops

L1
Deploy loop: Code → GitHub → Vercel/Render → live site. Push triggers auto-deploy; broken build? Rollback via Git history.
L2
Auth loop: Browser → Supabase Auth → JWT issued → all subsequent API calls carry token → RLS enforces data isolation per surveyor.
L3
Comparable engine loop: Frontend requests comps → Render orchestrates 4-tier geo search → queries Supabase + HMLR → applies hard deck filters → returns ranked results → surveyor selects → writes back to Supabase.
L4
Data enrichment loop: New case created → Render calls EPC, flood, broadband APIs → results cached in Supabase → frontend displays enriched data → surveyor validates.
L5
AI advisory loop: AI analyses comps + subject data → suggests adjustments & MV → surveyor reviews, overrides, confirms → final decision always with surveyor (RICS compliance).
L6
Dev iteration loop: Claude Code writes code locally → test in browser → fix issues → commit → deploy → user feedback → next iteration. Parallel Claude Code instances across different features.

⟩ Core Design Principle

AI advises — the platform automates data gathering, enrichment, and analysis.
Surveyor decides — every material judgement remains with the MRICS professional.
